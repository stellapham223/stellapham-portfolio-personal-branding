Case Study 03

Data Dashboard Authentication

Giải quyết cost spike 8x và lộ dữ liệu nhạy cảm bằng cách thiết kế hệ thống Google Auth + role-based access control cho internal analytics dashboard — giảm chi phí xuống 8 lần sau khi triển khai.

📌 Product + Security 🏢 Avada Group 🛠️ Vercel · Google Auth ✅ Shipped — cost reduced 8x

🔍 Context & Problem

Context

Dashboard analytics nội bộ được xây dựng trên Vercel, ban đầu chỉ phục vụ team sản phẩm. Sau lần update thứ 2 — bổ sung data Time to Value và North Star Metric — hệ thống đột ngột tăng chi phí đáng kể.

Trigger

Sau update thứ 2, chi phí nhảy từ ~$3.1/query lên $24/query — tăng gần 8 lần. Dashboard vẫn publicly accessible, external traffic và automated requests gây excessive data queries và cost escalation.

Core Issues Identified

  • Dashboard công khai truy cập, không có access control
  • External traffic và automated requests gây excessive queries
  • Dữ liệu nhạy cảm (revenue, North Star Metric) bị expose mà không có authorization
  • Không có role-based access management cho internal stakeholders
  • Real-time updates trigger frequent recomputation và data fetches không cần thiết

💡 Hypotheses

H1

Nếu giảm real-time updates xuống scheduled updates, chi phí hệ thống sẽ giảm đáng kể mà không ảnh hưởng đến chất lượng ra quyết định.

H2

Nếu thêm Google authentication, external spam và unauthorized access sẽ bị loại bỏ hoàn toàn.

H3

Nếu giới thiệu role-based access, data governance và internal collaboration sẽ được cải thiện đáng kể.

🏗️ Solution Design

7.1 System & Access Flow Architecture

Hai kiến trúc được cân nhắc: (a) Authentication sau khi UI load, (b) Authentication tại access-control layer trước khi fetch data. Phương án được chọn: Request interception và validation trước khi serve dashboard data + email-based allowlist check sau Google authentication.

7.2 Authentication & Authorization

  • Google Sign-In làm authentication method
  • Validate user email against an allowlist
  • Admin role: quản lý whitelist và approve access requests
  • Viewer role: read-only access
  • Unauthorized users → "Access Denied" screen + request access flow

7.3 UI & Admin Management

  • Login screen với Google authentication
  • Access denied và request access screens
  • Admin dashboard: quản lý whitelist (add/remove)
  • Assign roles và approve/reject access requests
  • Giảm phụ thuộc vào Engineering cho access management

🤝 Collaboration

Phối hợp với

  • Engineering: authentication flow, request interception, data update scheduling
  • Stakeholders: xác định ai nên có admin vs. viewer access

Trách nhiệm của tôi

  • Xác định access control rules và roles
  • Quyết định update frequency strategy
  • Căn chỉnh solution với cost, security, và usability goals

📈 Impact & Learnings

Business & Technical Outcomes

  • Infrastructure cost giảm và ổn định sau implementation
  • Unauthorized external access bị loại bỏ hoàn toàn
  • Sensitive revenue và product metrics được bảo mật

Operational Outcomes

  • Admin rights được delegate cho trusted users
  • Admins có thể tự quản lý: add/remove viewers, approve requests
  • Giảm phụ thuộc vào Engineering cho access management

Key Learning

Security và cost considerations phải được coi là product requirements, không phải technical afterthoughts. Not all analytics require real-time data to be valuable — đây là insight quan trọng giúp giải quyết cả 2 vấn đề cost và security cùng lúc.

Next Steps

  • Introduce finer-grained permissions by data type
  • Add audit logs for access changes
  • Consider SSO expansion if dashboard usage scales across teams
